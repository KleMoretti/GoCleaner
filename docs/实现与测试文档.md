# GoCleaner 实现与测试文档

> 版本：v1.0 草稿  
> 日期：2026-06-12  
> 阶段：第 10-11 天 Wails 前端联调

---

## 1. 本阶段目标

第 10-11 天目标是把已实现的 Go 扫描、清理、日志模块通过 Wails 绑定接入 React 桌面界面，形成一次可演示的闭环：

```text
扫描 -> 结果表格 -> 分类/风险筛选 -> 勾选 -> 清理确认 -> 执行清理 -> 结果反馈 -> 操作日志
```

本阶段不新增高风险深度清理能力，不做注册表、插件扫描、文件粉碎的新实现，只收口文件清理主流程的可演示性和失败可见性。

## 2. 已联调功能

| 功能点 | 实现说明 | 验收要点 |
| --- | --- | --- |
| 扫描入口 | 前端点击“开始扫描”调用 Wails `Scan()` | 扫描中按钮禁用并显示状态 |
| 结果表格 | 展示名称、分类、风险、大小、修改时间、来源规则、路径 | 高风险行有红色背景和高风险标签 |
| 分类筛选 | 支持全部分类、系统清理、软件缓存、隐私痕迹、插件扫描 | 筛选后表格和可见数量同步变化 |
| 风险筛选 | 支持全部风险、低风险、中风险、高风险 | 只影响当前展示，不修改扫描结果 |
| 勾选清理 | 支持单项勾选、选择当前可见安全项、清空选择 | 高风险不被“选择安全项”自动勾选 |
| 顶部统计 | 显示扫描文件数、扫描总大小、已勾选数、已勾选大小、失败项、高风险选中数 | 失败项合并展示扫描失败和清理失败数量 |
| 清理确认 | 清理前显示选中项、预计释放空间、风险构成 | 包含高风险时追加二次确认 |
| 清理结果 | 展示删除数量、释放空间、失败数量、失败路径和原因 | 失败项保留在表格中并取消勾选 |
| 操作日志 | 展示扫描/清理日志，支持展开失败详情 | 日志包含失败路径和失败原因 |

## 3. 演示流程

1. 启动 GoCleaner 桌面程序，确认顶部状态为“后端已连接”。
2. 点击“开始扫描”，等待扫描完成。
3. 查看顶部统计，确认显示扫描文件数、扫描总大小和失败项数量。
4. 在“扫描结果”表格中切换分类和风险筛选，确认表格数量随筛选变化。
5. 点击“选择当前可见安全项”，确认低/中风险可见项被选中，高风险项不被自动选中。
6. 点击“清理 N 项”，确认弹窗显示选中数量、预计释放空间和风险构成。
7. 确认清理后查看“清理结果”，确认成功删除项从表格移除，失败项仍可见并展示原因。
8. 查看“操作日志”，确认新增 scan / clean 记录，失败记录可展开查看路径和原因。

推荐演示数据使用 `%TEMP%` 下临时 `.tmp` / `.log` 文件，不使用真实系统关键目录做破坏性演示。

## 5. 风险控制说明

- 扫描先于清理：前端没有任何绕过扫描结果的直接删除入口。
- 预览先于删除：所有清理项均来自结果表格，用户可查看路径、来源规则、风险等级。
- 高风险默认不选：扫描器和规则加载层已保证高风险项默认不勾选，前端“选择当前可见安全项”也会排除高风险。
- 失败必须可见：扫描失败、清理失败、日志失败详情均展示路径和原因。
- 操作必须留痕：扫描和清理均写入 `data/operation.jsonl`，前端读取最新日志展示。
