# GoCleaner 实现与测试文档

> 版本：v1.0 草稿  
> 日期：2026-06-13  
> 阶段：第 14-17 天 注册表、文件粉碎、异常处理和体验完善

---

## 1. 本阶段目标

第 14-15 天实现两个独立的高风险演示功能：

```text
HKCU Run 无效启动项扫描 -> 注册表备份 -> 手动删除选中值
用户手动选择单个普通文件 -> 1/3/7 次覆写 -> flush/sync -> 随机重命名 -> 删除
```

这两个功能均不进入普通“一键清理”流程。注册表项和文件粉碎都必须通过独立入口触发，执行前显示风险提示并二次确认，失败原因写入结果和操作日志。

## 2. 已联调功能

| 功能点 | 实现说明 | 验收要点 |
| --- | --- | --- |
| 注册表扫描 | 新增 `ScanInvalidStartupRegistry()`，只读取 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | 不做全注册表扫描，不读取 HKLM |
| 启动命令解析 | 支持 `REG_SZ` / `REG_EXPAND_SZ`，解析引号路径、环境变量路径和参数 | 相对命令、系统命令、不明确目标不误报 |
| 注册表扫描项 | 返回 `type=registry`、`category=registry`、`risk=high`、`selected=false` | 高风险默认不勾选 |
| 注册表备份删除 | 新增 `DeleteRegistryItems(items, confirmed)` | 删除前写入 `data/registry_backup/*.reg`，备份失败拒绝删除 |
| 注册表失败反馈 | 删除成功/失败均返回路径、值名、备份路径和原因 | 前端结果面板和操作日志可见 |
| 文件选择 | 新增 `SelectShredFile()`，通过 Wails 文件选择器选单个文件 | 不支持批量扫描或目录粉碎 |
| 文件粉碎 | 新增 `ShredFile(request, confirmed)`，仅允许 1 / 3 / 7 次覆写 | 拒绝空路径、目录、符号链接、不存在路径和非法 passes |
| 粉碎流程 | 每次覆写后 `Sync()`，最后随机重命名并删除 | 成功返回粉碎数量和释放大小，失败返回原因 |
| 前端入口 | 增加“扫描注册表”“备份并删除注册表项”“选择文件粉碎” | 注册表和普通文件不能走同一个清理按钮 |
| 操作日志 | 新增 `registry_backup`、`registry_delete`、`shred` 日志 | 保留失败路径、失败原因和耗时 |

## 3. 风险提示文案

UI 和报告固定展示以下限制：

```text
注册表删除可能影响应用自启动。
文件粉碎是高风险操作。SSD、NTFS 日志、系统缓存、云同步目录等场景下无法保证专业取证级不可恢复。
```

注册表删除前会提示作用范围仅为 HKCU Run 无效启动项，并说明会先导出 `.reg` 备份。文件粉碎前会提示目标文件路径、覆写次数，以及粉碎后无法通过本程序恢复。

## 4. 演示流程

1. 启动 GoCleaner，点击“开始扫描”完成普通文件和插件扫描。
2. 点击“扫描注册表”，确认只新增“注册表”分类下的高风险项，且默认不勾选。
3. 勾选一个注册表项，点击“备份并删除注册表 N 项”，完成两次确认。
4. 查看“注册表结果”，确认展示删除数量、备份路径和失败原因。
5. 点击“选择文件粉碎”，选择测试目录中的普通文件。
6. 在覆写次数中选择 1 / 3 / 7，点击“确认粉碎”，完成两次确认。
7. 查看“粉碎结果”和“操作日志”，确认 `registry_backup`、`registry_delete`、`shred` 记录可见。

推荐演示数据使用测试注册表值和临时文件，不使用真实业务软件的启动项或重要文件。

## 6. 风险控制说明

- 扫描先于清理：注册表删除只能从扫描结果中选择项，文件粉碎只能手动选择单个文件。
- 预览先于删除：注册表项展示 hive、key、value、原始数据和目标路径；文件粉碎展示路径和覆写次数。
- 高风险默认不选：注册表扫描项固定 `risk=high` 且 `selected=false`。
- 注册表先备份再修改：`.reg` 备份写入失败时直接拒绝删除。
- 删除范围受限：注册表仅删除选中 value，不删除整个 key；不做全注册表扫描或自动修复。
- 粉碎范围受限：只处理普通文件，拒绝目录、符号链接、空路径和不存在路径。
- 失败必须可见：备份失败、删除失败、粉碎失败均返回路径和原因。
- 操作必须留痕：注册表备份、注册表删除、文件粉碎均写入 `data/operation.jsonl`。

## 7. 第 16-17 天体验完善记录

本阶段围绕异常可见性和演示体验做完善，不扩大清理范围，不新增无确认的深度清理能力。

| 功能点 | 实现说明 | 验收要点 |
| --- | --- | --- |
| 权限不足提示 | 扫描阶段将 `Access is denied` / `permission denied` 归类为“权限不足”，前端在扫描和处理结果中显示恢复建议 | 不静默忽略失败路径，失败原因进入结果面板和操作日志 |
| 扫描进度 | 新增 `ScanProgress` 事件载荷，通过 Wails 事件 `gocleaner:scan-progress` 推送加载规则、扫描文件、扫描插件、扫描注册表、完成等阶段 | UI 展示阶段、当前规则、发现项、失败项和近似百分比 |
| 清理确认弹窗 | 普通清理、插件隔离、注册表删除、文件粉碎均使用应用内确认弹窗 | 高风险普通清理、注册表删除和文件粉碎保留二次确认 |
| 空结果处理 | 未扫描、扫描后无项目、筛选后无匹配项使用不同文案 | 空结果不是错误，不引导用户执行危险操作 |
| 失败和部分成功 | 前端按成功数和失败数标记“全部成功 / 部分成功 / 全部失败 / 未处理任何项目” | 失败项保留在列表中并取消勾选，成功项从结果中移除 |

扫描进度为阶段式近似进度。程序不会为了精确百分比额外预扫描磁盘，避免增加扫描耗时和权限风险。
