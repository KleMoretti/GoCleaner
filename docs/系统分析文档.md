## 1. 引言

### 1.1 项目背景

GoCleaner 是一款面向 Windows 个人电脑的操作系统空间清理工具，基于 Go + Wails + React 技术栈实现。项目来源于"操作系统空间清理工具的设计与实现"课程任务书，目标是在 20 天实训周期内完成一个可运行、可演示、可测试、可写报告的空间清理工具。

本项目定位为课程实训可交付版本，不追求替代 360、火绒等商用清理软件，强调功能完整、风险可控、可演示、可测试、可写报告。

### 1.2 术语定义

| 术语 | 说明 |
| --- | --- |
| 扫描项 (ScanItem) | 一次扫描发现的单个可清理目标（文件、目录、注册表项、插件等） |
| 清理规则 (CleanRule) | JSON 配置文件中定义的一条清理策略，包含路径、模式、风险等级等 |
| 风险等级 | low / medium / high 三级，决定默认勾选行为和确认流程 |
| 隔离区 (Quarantine) | 高风险文件在删除前移动到的临时目录，支持手动恢复 |
| 文件粉碎 (Shred) | 通过覆写、重命名、删除等手段使文件难以恢复的操作 |
| 操作日志 (Operation Log) | JSONL 格式的操作记录，记录每次扫描和清理的详细信息 |

## 2. 任务书五类功能

根据任务书要求，GoCleaner 需覆盖以下五大功能类别：

### 2.1 系统日志、缓存、临时文件清理

**功能范围：**

| 子功能 | 目标路径 | 说明 |
| --- | --- | --- |
| 用户临时文件 | `%TEMP%` | 用户级临时目录，低风险 |
| 应用本地临时文件 | `%LOCALAPPDATA%\Temp` | 应用产生的临时文件 |
| 系统临时文件 | `C:\Windows\Temp` | 系统级临时目录，**高风险** |
| 系统日志文件 | `C:\Windows\Logs` | Windows 日志目录，**高风险** |
| 系统更新缓存 | `C:\Windows\SoftwareDistribution\Download` | Windows Update 下载缓存，**高风险** |

**清理策略：**
- 用户临时文件（`%TEMP%`、`%LOCALAPPDATA%\Temp`）：低风险，默认可选，支持按文件年龄过滤（如仅清理 7 天前的文件）
- 系统目录临时文件：高风险，默认不勾选，需要管理员权限，清理前需二次确认

### 2.2 软件产生的可清理文件清理

**功能范围：**

| 子功能 | 目标软件 | 清理范围 |
| --- | --- | --- |
| 浏览器缓存 | Chrome / Edge / Firefox | Cache、Code Cache、GPUCache、ShaderCache |
| 即时通讯缓存 | QQ / 微信 | 日志、缓存、临时文件（**禁止**清理聊天记录、图片、文件、数据库） |
| 其他软件缓存 | 通过 JSON 规则扩展 | 按规则文件配置的路径和模式匹配 |

**清理策略：**
- 浏览器缓存：低风险，默认可选，仅清理缓存目录，不触碰 History、Cookies、Login Data、Web Data、Preferences 等敏感数据
- QQ / 微信：仅允许通过规则文件处理日志、缓存、临时文件，规则中必须明确标记排除项
- 扩展软件：通过 JSON 规则文件配置，支持路径、文件模式、排除列表和风险等级

### 2.3 隐私痕迹和无效注册表处理

#### 2.3.1 隐私痕迹清理

| 子功能 | 目标 | 风险等级 |
| --- | --- | --- |
| 最近文档记录 | `%APPDATA%\Microsoft\Windows\Recent` | medium |
| 跳转列表 | `%APPDATA%\Microsoft\Windows\Recent\AutomaticDestinations` | medium |
| 缩略图缓存 | `%LOCALAPPDATA%\Microsoft\Windows\Explorer\thumbcache_*.db` | low |
| 浏览器缓存 | Chrome / Edge / Firefox 缓存目录 | low |
| 浏览器下载历史 | 浏览器的 History / Downloads 数据库 | **禁止清理** |
| 浏览器密码和 Cookie | Login Data、Cookies、Web Data | **禁止清理** |

**清理策略：**
- 隐私痕迹默认标记为 medium 风险，需要用户主动勾选
- 不得删除密码、Cookie、历史记录数据库文件
- 如未来支持这些敏感数据，必须放入高风险分类，并提供单独确认说明

#### 2.3.2 无效注册表处理

| 子功能 | 目标路径 | 说明 |
| --- | --- | --- |
| 无效启动项 | `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | 检测指向不存在文件的启动项 |
| 最近文档引用 | `HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs` | 检测无效文件引用 |
| 无效卸载项 | `HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall` | 检测指向不存在路径的卸载信息（可选扩展） |

**注册表操作约束（必须）：**

```text
1. 仅扫描明确列出的安全路径
2. 默认只展示问题项（指向不存在文件的值），不展示有效项
3. 删除前导出 .reg 备份文件到 data/registry_backup/
4. 删除前 UI 二次确认弹窗
5. 删除失败必须展示具体原因（权限不足、键被锁定等）
```

**禁止实现：** 全注册表扫描、全自动注册表修复、一键深度注册表清理。

### 2.4 不必要插件扫描或清理

**功能范围：**

| 子功能 | 目标 | 说明 |
| --- | --- | --- |
| Chrome 扩展扫描 | `%LOCALAPPDATA%\Google\Chrome\User Data\*\Extensions` | 读取 manifest.json 获取扩展名和版本 |
| Edge 扩展扫描 | `%LOCALAPPDATA%\Microsoft\Edge\User Data\*\Extensions` | 读取 manifest.json 获取扩展名和版本 |
| 插件信息展示 | 扩展名、版本、路径、大小 | 表格展示，不直接删除 |

**清理策略：**
- 默认仅扫描和展示，不执行删除
- 可选实现"移动到隔离区"功能，将插件目录移至 `data/quarantine/`，支持手动恢复
- 不实现自动判断"不必要插件"的启发式逻辑（容易误判），由用户自行判断

### 2.5 垃圾文件粉碎

**功能范围：**

| 子功能 | 说明 |
| --- | --- |
| 文件选择 | 用户手动指定要粉碎的文件（非批量自动粉碎） |
| 覆写粉碎 | 分块覆写文件内容（可选覆写次数 1/3/7 次） |
| 刷盘同步 | 每次覆写后调用 `FlushFileBuffers` 刷盘 |
| 随机重命名 | 覆写后随机重命名文件 |
| 最终删除 | 重命名后删除文件 |
| 结果反馈 | 展示粉碎成功/失败状态和原因 |

**技术限制说明（必须在 UI 或报告中声明）：**

```text
1. SSD 写入放大和磨损均衡：物理扇区可能未被实际覆写
2. NTFS 日志（$LogFile）：可能残留文件内容副本
3. 系统还原和卷影复制：可能保留文件早期版本
4. 云同步目录（OneDrive 等）：云端可能保留副本
5. 文件系统缓存和预读缓存：可能暂存文件内容
```

本项目实现的文件粉碎为**教学意义上的覆写删除**，不能保证专业取证级不可恢复。

## 3. 核心安全原则

### 3.1 "默认安全、手动确认、高风险不自动清理"原则

GoCleaner 所有清理操作遵循以下铁律：

#### 3.1.1 默认安全 (Safe by Default)

```text
1. 低风险项 (low)：默认勾选，用户可直接清理
   - 例：用户 %TEMP% 目录中的临时文件
   - 例：浏览器 GPUCache、ShaderCache

2. 中风险项 (medium)：默认展示但不勾选，用户需主动选择
   - 例：Recent 文档记录、JumpList
   - 例：缩略图缓存

3. 高风险项 (high)：默认不勾选，带有明显的高风险标识，需要额外确认
   - 例：C:\Windows\Temp 系统临时文件
   - 例：注册表无效项
   - 例：系统更新缓存
```

#### 3.1.2 手动确认 (Manual Confirmation)

```text
1. 普通清理（仅含低风险项）→ 一次确认弹窗
2. 含中风险项清理 → 弹窗中列出中风险项，用户确认
3. 含高风险项清理 → 额外二次确认弹窗，明确说明风险
4. 注册表清理 → 确认 + 备份提示 + 二次确认
5. 文件粉碎 → 确认 + 不可恢复警告
```

#### 3.1.3 高风险不自动清理 (No Auto-Clean for High Risk)

```text
1. 高风险项绝不出现在任何"一键清理"流程中
2. 高风险项默认不被选中，必须用户逐项或逐类手动勾选
3. 程序启动时不执行任何自动清理
4. 不得提供"全选高风险项"按钮
5. 不得实现没有确认流程的一键深度清理
```

### 3.2 操作铁律

```text
1. 安全优先           — 任何功能设计首先考虑用户数据安全
2. 扫描先于清理        — 永远先扫描展示结果，再执行清理
3. 预览先于删除        — 用户必须能看到将要删除的内容
4. 高风险默认不选      — 高风险项默认不被勾选
5. 注册表先备份再修改   — 每次修改注册表前导出 .reg 备份
6. 失败必须可见        — 清理失败的文件必须展示路径和原因
7. 操作必须留痕        — 所有扫描和清理行为记录到操作日志
```

## 4. 风险等级定义

### 4.1 三级风险体系

| 等级 | 标识 | 默认行为 | 确认要求 | 示例 |
| --- | --- | --- | --- | --- |
| **low** | 绿色 | 默认勾选 | 统一确认即可 | 用户 TEMP、浏览器缓存 |
| **medium** | 黄色 | 展示不勾选 | 弹窗列出中风险项 | Recent、JumpList、缩略图 |
| **high** | 红色 | 不勾选 | 二次确认弹窗 | 系统目录、注册表、粉碎 |

### 4.2 风险等级判定规则

```text
low   → 路径在用户目录下 + 明确是临时/缓存文件 + 删除不影响系统或应用正常运行
medium → 涉及隐私痕迹 或 路径在系统盘但非关键目录
high  → 涉及系统目录、注册表、可能影响系统稳定性 或 需要管理员权限
```

### 4.3 高风险目录清单

以下路径始终标记为高风险（除非规则文件显式覆盖为 medium）：

```text
C:\Windows\Temp
C:\Windows\Logs
C:\Windows\SoftwareDistribution\Download
C:\Windows\Prefetch
C:\Windows\Minidump
C:\Windows\System32\config
C:\Windows\System32\winevt\Logs
HKCU\Software\*   （所有注册表路径）
HKLM\Software\*   （所有注册表路径）
```

### 4.4 禁止清理清单（硬编码）

以下路径和数据在任何情况下不得清理：

```text
# 浏览器敏感数据
*:\History
*:\Cookies
*:\Login Data
*:\Web Data
*:\Preferences           （除非用户明确要求且放入高风险分类）

# QQ / 微信核心数据
*:\Tencent Files\*\Audio
*:\Tencent Files\*\Image
*:\Tencent Files\*\File
*:\Tencent Files\*\Video
*:\WeChat Files\*\File
*:\WeChat Files\*\Image
*:\WeChat Files\*\Video
*:\Msg3.0.db
*:\Msg.db

# 系统关键目录
C:\Windows\System32
C:\Windows\SysWOW64
C:\Windows\Boot
C:\Windows\Installer
C:\Program Files
C:\Program Files (x86)

# 项目自身目录
<GoCleaner 安装目录>\*
```

## 6. 核心流程

### 6.1 扫描流程

```text
用户点击"开始扫描"
        │
        ▼
┌───────────────────┐
│ 加载规则文件        │ ← cleaner_rules.json
│ 校验规则合法性      │
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│ 展开环境变量        │ ← %TEMP% → C:\Users\xxx\AppData\Local\Temp
│ 检查路径存在性      │
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│ 按规则遍历目录      │
│ 匹配文件模式        │ ← *.log, *.tmp, *.dmp...
│ 排除白名单路径      │
│ 按 MinAgeDays 过滤  │
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│ 计算文件大小        │
│ 设置风险等级        │ ← 来自规则定义
│ 设置默认选中状态    │ ← 来自规则定义（受风险等级约束）
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│ 汇总返回 ScanItem[] │
│ 记录扫描日志        │
└───────────────────┘
```

### 6.2 清理流程

```text
用户在结果表格中勾选要清理的项
        │
        ▼
┌──────────────────────────┐
│ 统计选中项                   │
│ ├─ 低风险项数量和大小         │
│ ├─ 中风险项数量和大小         │
│ └─ 高风险项数量和大小         │

       │
       ▼
┌──────────────────────────┐
│ 根据最高风险等级生成确认弹窗   │
│ ├─ 仅低风险 → 普通确认弹窗    │
│ ├─ 含中风险 → 列出中风险项    │
│ └─ 含高风险 → 额外二次确认    │
└──────┬───────────────────┘
       │ 用户确认
       ▼
┌──────────────────────────┐
│ 重新校验文件是否存在         │
│ (文件可能已被其他程序删除)    │
└──────┬───────────────────┘
       │
       ▼
┌──────────────────────────┐
│ 逐项执行删除                │
│ ├─ 成功 → 记录释放空间      │
│ ├─ 被占用 → 记录失败原因    │
│ └─ 权限不足 → 记录失败原因  │
└──────┬───────────────────┘
       │
       ▼
┌──────────────────────────┐
│ 返回 CleanResult           │

│ 刷新 UI 展示结果            │
└──────────────────────────┘
```

## 7. 非功能需求

### 7.1 性能要求

| 指标 | 要求 |
| --- | --- |
| 扫描速度 | 10 万文件以内目录 ≤ 30 秒完成扫描 |
| UI 响应 | 扫描期间 UI 不卡死，支持取消操作 |
| 内存占用 | 空闲 ≤ 100MB，扫描中 ≤ 500MB |
| 安装包大小 | ≤ 50MB |

### 7.2 兼容性

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10 1809+ / Windows 11 |
| 架构 | amd64（x64） |
| 权限 | 支持普通用户权限运行，高风险功能提示管理员权限 |

### 7.3 可靠性

| 场景 | 处理方式 |
| --- | --- |
| 规则文件缺失 | 使用内置默认规则，记录警告日志 |
| 规则格式错误 | 跳过该条规则，记录错误详情，不崩溃 |
| 路径不存在 | 跳过该路径，不报错 |
| 文件被占用 | 跳过该文件，记录失败原因 |
| 权限不足 | 跳过该文件/注册表项，记录失败原因 |
| 磁盘空间不足 | 停止当前操作，提示用户 |

### 7.4 安全性

| 要求 | 说明 |
| --- | --- |
| 不修改系统关键文件 | 硬编码禁止路径清单 |
| 不联网 | 程序完全离线运行 |
| 不上传用户数据 | 不收集任何用户信息 |
| 操作可审计 | 所有操作有日志可查 |

### 7.5 可维护性

| 要求 | 说明 |
| --- | --- |
| JSON 规则扩展 | 新增清理目标只需修改规则文件，无需改代码 |
| 模块化架构 | scanner / cleaner / rules / windows / logger 各司其职 |
| 代码规范 | gofmt / go vet / go test |

## 8. 数据字典

### 8.1 CleanRule（清理规则）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| Name | string | 是 | 规则名称，如 "Chrome Cache" |
| Category | string | 是 | 分类：system / software / privacy / plugin |
| Paths | []string | 是 | 扫描路径，支持环境变量如 `%LOCALAPPDATA%\Google\Chrome\User Data\*\Cache` |
| Patterns | []string | 否 | 文件匹配模式，如 `["*.log", "*.tmp"]`。为空则匹配所有文件 |
| Exclude | []string | 否 | 排除的子路径或文件模式 |
| Risk | string | 是 | 风险等级：low / medium / high |
| MinAgeDays | int | 否 | 最小文件年龄（天），0 表示不限制 |
| DefaultOn | bool | 是 | 默认是否勾选（受风险等级约束，高风险强制 false） |

### 8.2 ScanItem（扫描项）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| ID | string | 唯一标识，格式：`{category}_{path_hash}` |
| Path | string | 文件或注册表项的绝对路径 |
| Name | string | 文件名或注册表值名 |
| Type | string | 类型：file / directory / registry / plugin |
| Category | string | 分类 |
| Size | int64 | 文件大小（字节），目录为 0 |
| Risk | string | 风险等级：low / medium / high |
| Source | string | 来源规则名称 |
| LastModified | int64 | 最后修改时间（Unix 时间戳） |
| Selected | bool | 当前是否被用户选中 |

### 8.3 CleanResult（清理结果）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| DeletedFiles | int | 成功删除的文件数 |
| FreedSize | int64 | 释放的磁盘空间（字节） |
| FailedFiles | []string | 删除失败的文件路径列表 |
| FailedReasons | map[string]string | 失败路径 → 失败原因 |
| Message | string | 结果摘要信息 |

### 8.4 OperationLog（操作日志）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| Timestamp | string | ISO 8601 时间戳 |
| Operation | string | scan / clean / shred / registry_backup / registry_delete |
| ScannedFiles | int | 扫描文件总数（scan 操作） |
| DeletedFiles | int | 删除文件数（clean 操作） |
| FreedSize | int64 | 释放空间（字节） |
| FailedPaths | []string | 失败路径列表 |
| FailedReasons | []string | 失败原因列表 |
| Duration | int64 | 操作耗时（毫秒） |

## 11. 风险与约束

### 11.1 技术风险

| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| Windows API 调用复杂度 | 注册表、文件权限操作可能遇到兼容性问题 | 封装在 internal/windows，统一处理错误 |
| Wails 框架学习曲线 | 初期联调效率 | 先用 Go 核心逻辑跑通测试，再联调 UI |
| SSD 粉碎不可靠 | 粉碎效果无法验证 | 文档和 UI 中明确声明限制 |
| 管理员权限 | 部分目录无法访问 | 检测权限，提示用户以管理员运行 |

### 11.2 范围约束

```text
1. 不做全注册表扫描和自动修复
2. 不实现没有确认的一键深度清理
3. 不实现浏览器密码/Hash 提取
4. 不清理 QQ/微信聊天记录和核心数据
5. 不实现文件恢复（UnDelete）功能
6. 不实现定时自动清理
7. 不联网、不上传、不收集数据
```
